# Code-quality sweep — 2026-05

Companion to `audit-2026-05.md` (the security toolchain run).
Findings here are non-security code-quality items.

## Fixed in this PR

- **Dead variable: `for_count`** in
  `backend/app/services/playback/stats.py:_apply_filters`. Caller
  passed `for_count=True`; the parameter was never read inside the
  function body. Removed from both signature and call site.

- **Unused frontend exports** flagged by `knip`:
  - `useTagNames` in `hooks/useTags.ts` — added preemptively
    alongside the tag-management PR; no callers landed.
  - `fmtTime` in `lib/format.ts` — a different `fmtTime` is defined
    locally in `LogsPage.tsx`; the lib version has no callers.
  - `HelpButton` in `components/shell/HelpDrawer.tsx` — exported
    but never imported.

- **`@eslint/js` undeclared dep** — used in `eslint.config.js:1`
  but missing from `package.json`. Worked locally because it's a
  transitive of `eslint`; declaring it explicitly so a `npm ci` on
  a slim install reliably resolves it.

## Follow-ups (not addressed in this PR)

### `ruff` backlog (263)

Almost entirely cosmetic; `make format` candidates. Largest buckets:

- `RUF100` × 84 — unused `# noqa` comments.
- `UP037` × 30 — outdated typing syntax (`Optional[X]` → `X | None`).
- `I001` × 27 — import order.
- `RUF012` × 17 — mutable class defaults.
- `RUF022` × 13 — `__all__` not sorted.

A single `uv run ruff check --fix` would clear most of them but the
diff is too large to land in a quality PR.

### `ASYNC240` — sync filesystem calls in async functions (9)

Mostly in `app/optimization/{ffmpeg_runner.py,worker.py}`. The
worker is single-tenant per tick (one ffmpeg job at a time), so a
brief block isn't fatal, but a slow NFS share or a 4K transcode's
output size check (`os.path.getsize`) can stall the worker. Swap
to `anyio.Path` in a follow-up.

### Vestigial "Stage N removed" comments (~64)

Files like `app/api/v1/media.py`, `app/models/media.py`,
`app/services/repositories/media.py` carry chunks of comments
describing the pre-Stage-05 quarantine API surface that was
retired. Useful as history in PRs but redundant in source. A
sweep pass could drop the ~64 lines without losing context that
`git blame` doesn't already preserve.

### `usePluginRegistry` family (frontend)

`plugins/registry.ts` defines `usePluginRegistry`,
`usePluginWidgets`, `usePluginSettings` as part of a plugin SDK
that hasn't been picked up by any plugin yet. They're intentional
framework exports — leave alone but consider adding to a knip
ignore list so future runs stay clean.

### Frontend dev-dep advisories (npm audit)

5 moderate, all in vite/vitest/esbuild dev-only deps. Dependabot
will bump on the next weekly cycle.
