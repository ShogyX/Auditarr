## Findings

### Fixed in this audit

1. **B324 / S324 ‚Äî SHA-1 hashing flagged as "for security"** ‚Äî
   `app/services/ai/suggestions.py:684` and
   `app/services/playback/poller.py:1008`. Both call sites use the
   short digest as a stable dedup-key, not as a signature. Added
   `usedforsecurity=False` per CPython's documented contract so the
   linter understands the intent.

2. **B701 / S701 ‚Äî Jinja2 environment with `autoescape=False`** ‚Äî
   `app/notifications/templating.py`. Current consumers (SMTP
   `text_body`, JSON webhook payloads) don't HTML-render the
   output, so the finding wasn't exploitable. Flipped to
   `select_autoescape(("html", "htm", "xml"))` as defence-in-depth
   so a future plugin that decides to set `html_body` from the same
   template can't silently regress into an XSS.

3. **`use-defused-xml`** ‚Äî `plugins/plex/backend.py` used
   `xml.etree.ElementTree.fromstring` against Plex server responses.
   Modern CPython is XXE-safe by default, but the parser is still
   vulnerable to billion-laughs / quadratic-blowup DoS. Swapped to
   `defusedxml.ElementTree` (now a hard dep in `pyproject.toml`).

### Secrets sweep

* `detect-secrets` flagged ~95 files ‚Äî every match outside
  `.env.example`, `alembic.ini` (placeholder), and `settings.py`
  default URL was either:
  * a test fixture password (`PASSWORD = "supersecret-password-1!"`)
  * a generated hex digest in cache dirs
  * docs showing operators which env vars to set
* `backend/.env` is correctly gitignored ‚Äî confirmed via
  `git log -- backend/.env` (no commits) and the GitHub Contents
  API (404 on `?ref=main`).
* Git history sweep for `BEGIN PRIVATE KEY`, `ghp_*`, and
  high-entropy `SECRET_KEY=` / `PASSWORD=` assignments: clean
  except the test fixture password.
